# v0.6.15

This patch release tightens Feishu topic routing and makes the Chat list behavior consistent with the newer project UI.

## Changes

- Routed Feishu topic messages such as `/chats ...` to the bound Codex thread when they are normal user input, while keeping real topic commands available.
- Prevented ordinary Feishu thread replies from repeatedly posting the "Codex thread topic opened" activation notice.
- Made `/chats` render the same green Chats project list used from `/projects`, including pinned chat styling and the temporary chat creation entry.
- Fixed Chat selection from the bot DM so it opens and binds a Feishu topic instead of continuing to render cards in the DM.
- Kept macOS release binaries signed with the stable `tech.mideco.codex-feishu.ctr-go` identifier for more predictable Accessibility authorization.
