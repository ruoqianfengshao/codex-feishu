# v0.6.5

This patch release fixes Feishu topic message routing.

## Changes

- Restores routing for Feishu topic replies when the incoming event resolves to
  a non-zero topic root while existing panel and message route records were
  stored under the legacy `topic_id=0` key.
- Adds a regression test for rootless/legacy Feishu topic route fallback so
  topic messages do not incorrectly fall back to the workspace "no Codex thread
  selected" hint.
