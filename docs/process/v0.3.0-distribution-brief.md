# v0.3.0 Distribution Brief

## Summary

`v0.3.0` improves first-run adoption by adding release binaries and `ctr-go init`. It does not change the Telegram runtime contract; the goal is to make the existing local-first daemon easier to install and configure safely.

## Target Behavior

- GitHub Releases provide archives for macOS arm64/amd64, Linux arm64/amd64, and Windows amd64.
- `ctr-go init` writes a private local config file and prints only safe next steps.
- `config.env` values are read automatically, while explicit environment variables keep priority.
- README and wiki Quickstart lead with the binary + init path and keep source builds as an alternative.

## Implementation Cycle

1. Add tests for config file loading and init behavior.
2. Implement the smallest config/init surface.
3. Add release packaging and documentation.
4. Run targeted config/CLI tests, full tests, build, release dry-run, diff check, and secret scan.
5. Validate the built binary with a private config path before tagging.

## Out Of Scope

- No automatic daemon/service installer.
- No hosted update channel.
- No App Server protocol changes.
- No Telegram UI/routing behavior changes.
